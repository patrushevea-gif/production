# Мини-инструкция для ИБ: принцип формирования пароля

## Назначение
Проект «Создание надежного пароля» генерирует пароль **на основе пользовательского ввода** (seed) без заранее подготовленных паролей в коде.

## Как работает алгоритм
1. **Пользовательский ввод**
   - Пользователь вводит число, слово или фразу.
2. **Нормализация ввода**
   - Удаляются пробелы по краям.
   - Текст приводится к нижнему регистру.
3. **Хеширование в seed**
   - Нормализованный текст преобразуется во внутреннее 32-битное число (seed) через hash-функцию.
4. **Псевдослучайная последовательность**
   - Seed подается в PRNG на базе xorshift.
   - PRNG выдает последовательность чисел для выбора символов.
5. **Сборка пароля по правилам сложности**
   - Длина пароля: **14 символов**.
   - Гарантированно включаются:
     - минимум **2 заглавные** латинские буквы,
     - минимум **2 специальных символа**,
     - строчные латинские буквы (и частично цифры для усиления вариативности).
6. **Перемешивание**
   - Символы перемешиваются (Fisher–Yates), чтобы убрать предсказуемые позиции.
7. **Вывод**
   - Пароль показывается пользователю и может быть скопирован кнопкой.

## Что важно для объяснения ИБ
- В коде **нет массива “готовых паролей”**.
- Результат зависит только от пользовательского ввода и алгоритма.
- При одинаковом вводе и одинаковой версии алгоритма результат одинаковый (детерминированность).
- При изменении ввода формируется другой пароль.
- Генерация выполняется локально в браузере, без отправки seed на сервер (в рамках текущей реализации страницы).
